## Backend архитектура

### Слои

```
┌─────────────────────────────────────────────────────────────┐
│                         Routes                               │
│            Определение HTTP маршрутов                        │
│            Привязка к контроллерам                           │
├─────────────────────────────────────────────────────────────┤
│                      Controllers                             │
│     Обработка HTTP запросов                                  │
│     Валидация входных данных                                 │
│     Формирование ответов                                     │
│     Обработка ошибок                                         │
├─────────────────────────────────────────────────────────────┤
│                        Services                              │
│     Бизнес-логика                                            │
│     Работа с хранилищем данных                               │
│     Взаимодействие между сущностями                          │
├─────────────────────────────────────────────────────────────┤
│                        Models                                │
│     TypeScript интерфейсы                                    │
│     Описание структуры данных                                │
├─────────────────────────────────────────────────────────────┤
│                  Data Storage (JSON Files)                   │
│     Файловое хранилище данных                                │
│     users.json, products.json, carts.json, orders.json       │
└─────────────────────────────────────────────────────────────┘
```

### Аутентификация

```
┌──────────────────────────────────────────────────────────────┐
│                    Регистрация / Вход                         │
│                                                               │
│  1. Валидация данных                                          │
│  2. Проверка уникальности (email, login)                      │
│  3. Хеширование пароля (bcrypt)                               │
│  4. Создание пользователя / проверка пароля                   │
│  5. Создание сессии (token + userId + expiresAt)              │
│  6. Установка httpOnly cookie (sessionToken)                  │
│                                                               │
└───────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│                    Защищённые запросы                         │
│                                                               │
│  1. authMiddleware извлекает токен из cookie                  │
│  2. SessionService проверяет валидность сессии                │
│  3. userId добавляется в request                              │
│  4. Контроллер обрабатывает запрос                            │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

**Длительность сессии:** 10 минут (автоматическое продление при активности)

## Хранение данных

### JSON файлы

| Файл | Структура |
|------|-----------|
| `users.json` | `User[]` - пользователи |
| `products.json` | `Product[]` - продукты |
| `carts.json` | `Cart[]` - корзины пользователей |
| `orders.json` | `Order[]` - заказы |
| `sessions.json` | `Session[]` - сессии авторизации |

### Client-side хранение

| Хранилище | Данные |
|-----------|--------|
| Memory (Store) | Состояние приложения, данные пользователя |
| localStorage | Тема (dark/light) |
| Cookies | sessionToken (httpOnly) |

## Безопасность

### Реализовано

- **httpOnly cookies** - защита от XSS атак на сессию
- **sameSite: lax** - защита от CSRF
- **bcrypt** - безопасное хеширование паролей
- **Валидация** - на frontend и backend
- **CORS** - ограничение origins в production

### Планируется

- HTTPS в production
- CSRF токены для критичных операций
- Rate limiting
- CAPTCHA при регистрации

## Запуск и окружение

### Development

```bash
# Backend (порт 3001)
npm run dev:backend

# Frontend (порт 3002)
npm run dev:frontend

# Одновременно
npm run dev
```

### Production

```bash
# Сборка
npm run build

# Запуск backend
npm start
```

### Переменные окружения

```env
# Backend
PORT=3001
NODE_ENV=development
FRONTEND_URL=http://localhost:3002

# Frontend
VITE_API_URL=http://localhost:3001
FRONTEND_PORT=3002
```
