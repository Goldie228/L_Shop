## 1. Аутентификация и управление аккаунтом

### UC-1.1: Регистрация нового пользователя

**Цель:** Создать новый аккаунт для доступа к функциям магазина.

**Акторы:** Гость (неавторизованный пользователь)

**Предусловия:**
- Пользователь не авторизован в системе

**Основной поток:**
1. Пользователь нажимает кнопку "Регистрация" в шапке сайта
2. Открывается модальное окно регистрации
3. Пользователь заполняет поля:
   - Имя (обязательно)
   - Email (обязательно, валидный формат)
   - Логин (обязательно, уникальный)
   - Телефон (обязательно, формат +375XXXXXXXXX)
   - Пароль (обязательно, минимум 6 символов)
4. Пользователь нажимает "Зарегистрироваться"
5. Система валидирует данные
6. Система проверяет уникальность email и логина
7. Система создаёт пользователя и сессию
8. Система авторизует пользователя
9. Модальное окно закрывается
10. Отображается приветственное сообщение

**Альтернативные потоки:**

**1.1a: Email уже существует**
- Система показывает ошибку "Пользователь с таким email уже существует"
- Пользователь может изменить email или войти в существующий аккаунт

**1.1b: Логин уже занят**
- Система показывает ошибку "Логин уже занят"
- Пользователь выбирает другой логин

**1.1c: Невалидный телефон**
- Система показывает ошибку "Неверный формат телефона"
- Пользователь исправляет номер

**1.1d: Слабый пароль**
- Система показывает требования к паролю
- Пользователь вводит более надёжный пароль

**Постусловия:**
- Создан новый пользователь
- Создана сессия (httpOnly cookie)
- Пользователь авторизован

---

### UC-1.2: Вход в систему

**Цель:** Авторизоваться в существующем аккаунте.

**Акторы:** Гость (неавторизованный пользователь)

**Предусловия:**
- Пользователь имеет зарегистрированный аккаунт
- Пользователь не авторизован в текущей сессии

**Основной поток:**
1. Пользователь нажимает кнопку "Войти" в шапке сайта
2. Открывается модальное окно входа
3. Пользователь вводит логин или email
4. Пользователь вводит пароль
5. Пользователь нажимает "Войти"
6. Система проверяет учётные данные
7. Система создаёт новую сессию
8. Модальное окно закрывается
9. Отображается имя пользователя в шапке

**Альтернативные потоки:**

**1.2a: Неверные учётные данные**
- Система показывает ошибку "Неверный логин или пароль"
- Пользователь может повторить попытку

**1.2b: Аккаунт заблокирован (будущее)**
- Система показывает сообщение о блокировке
- Предлагается связаться с поддержкой

**Постусловия:**
- Создана сессия
- Пользователь авторизован

---

### UC-1.3: Выход из системы

**Цель:** Завершить сессию и выйти из аккаунта.

**Акторы:** Авторизованный пользователь

**Предусловия:**
- Пользователь авторизован

**Основной поток:**
1. Пользователь нажимает на своё имя в шапке
2. Открывается выпадающее меню
3. Пользователь нажимает "Выйти"
4. Система удаляет сессию
5. Система очищает cookie
6. Пользователь перенаправляется на главную страницу
7. Отображаются кнопки "Войти" и "Регистрация"

**Постусловия:**
- Сессия удалена на сервере
- Cookie удалена
- Пользователь разавторизован

---

### UC-1.4: Восстановление пароля (будущее)

**Цель:** Восстановить доступ к аккаунту при утере пароля.

**Акторы:** Гость (неавторизованный пользователь)

**Предусловия:**
- Пользователь имеет зарегистрированный аккаунт
- Пользователь не помнит пароль

**Основной поток:**
1. Пользователь нажимает "Забыли пароль?" в форме входа
2. Открывается форма восстановления пароля
3. Пользователь вводит email
4. Система отправляет письмо со ссылкой для сброса пароля
5. Пользователь переходит по ссылке из письма
6. Пользователь вводит новый пароль
7. Система обновляет пароль
8. Пользователь автоматически входит в систему

**Альтернативные потоки:**

**1.4a: Email не найден**
- Система показывает сообщение "Если аккаунт существует, письмо отправлено"
- Не раскрывает существование аккаунта (безопасность)

**1.4b: Ссылка истекла**
- Система показывает ошибку "Ссылка истекла"
- Предлагает запросить новую ссылку

### UC-1.5: Изменение пароля (будущее)

**Цель:** Изменить текущий пароль на новый.

**Акторы:** Авторизованный пользователь

**Предусловия:**
- Пользователь авторизован

**Основной поток:**
1. Пользователь переходит в настройки профиля
2. Пользователь нажимает "Изменить пароль"
3. Пользователь вводит текущий пароль
4. Пользователь вводит новый пароль дважды
5. Система проверяет текущий пароль
6. Система обновляет пароль
7. Отображается сообщение об успехе

**Альтернативные потоки:**

**1.5a: Неверный текущий пароль**
- Система показывает ошибку
- Пользователь повторяет попытку

---

### UC-1.6: Редактирование профиля (будущее)

**Цель:** Изменить персональные данные.

**Акторы:** Авторизованный пользователь

**Предусловия:**
- Пользователь авторизован

**Основной поток:**
1. Пользователь переходит в профиль
2. Пользователь нажимает "Редактировать"
3. Пользователь изменяет данные:
   - Имя
   - Телефон
   - Email (с подтверждением)
4. Пользователь сохраняет изменения
5. Система валидирует и обновляет данные

---

### UC-1.7: Удаление аккаунта (будущее)

**Цель:** Полностью удалить аккаунт и данные.

**Акторы:** Авторизованный пользователь

**Предусловия:**
- Пользователь авторизован
- У пользователя нет активных заказов

**Основной поток:**
1. Пользователь переходит в настройки аккаунта
2. Пользователь нажимает "Удалить аккаунт"
3. Система запрашивает подтверждение паролем
4. Система показывает предупреждение о потере данных
5. Пользователь подтверждает удаление
6. Система помечает аккаунт для удаления
7. Пользователь выходит из системы

**Постусловия:**
- Аккаунт деактивирован
- Персональные данные удалены согласно GDPR
- История заказов сохранена анонимно

---

### UC-1.8: Смена email (будущее)

**Цель:** Изменить привязанный email адрес.

**Акторы:** Авторизованный пользователь

**Основной поток:**
1. Пользователь запрашивает смену email
2. Пользователь вводит новый email
3. Система отправляет подтверждение на старый email
4. Пользователь подтверждает на старом email
5. Система отправляет подтверждение на новый email
6. Пользователь подтверждает на новом email
7. Email обновлён

---

### UC-1.9: Подключение социальной аутентификации (будущее)

**Цель:** Войти через Google/Apple/VK.

**Акторы:** Гость или авторизованный пользователь

**Основной поток:**
1. Пользователь нажимает "Войти через Google"
2. Открывается OAuth страница провайдера
3. Пользователь авторизуется у провайдера
4. Провайдер возвращает токен
5. Система создаёт/находит аккаунт
6. Пользователь авторизован в L_Shop

**Альтернативные потоки:**

**1.9a: Email уже привязан к другому аккаунту**
- Система предлагает войти в существующий аккаунт
- Или привязать социальную сеть к текущему аккаунту

---

### UC-1.10: Двухфакторная аутентификация (будущее)

**Цель:** Повысить безопасность аккаунта.

**Акторы:** Авторизованный пользователь

**Основной поток:**
1. Пользователь включает 2FA в настройках
2. Система показывает QR-код для приложения-аутентификатора
3. Пользователь сканирует код
4. Пользователь вводит код для подтверждения
5. 2FA включён

**При входе:**
1. После ввода пароля запрашивается код
2. Пользователь вводит код из приложения
3. Доступ предоставлен

## 2. Каталог и поиск продуктов

### UC-2.1: Просмотр каталога продуктов

**Цель:** Найти и изучить доступные товары.

**Акторы:** Любой пользователь

**Предусловия:**
- Каталог содержит минимум один продукт

**Основной поток:**
1. Пользователь переходит на главную страницу
2. Отображается список продуктов с:
   - Изображением
   - Названием
   - Ценой
   - Скидкой (если есть)
   - Статусом наличия
3. Пользователь может перейти на следующую страницу (пагинация)

**Постусловия:**
- Пользователь видит доступные товары

---

### UC-2.2: Фильтрация продуктов по категории (будущее)

**Цель:** Найти товары определённой категории.

**Акторы:** Любой пользователь

**Основной поток:**
1. Пользователь выбирает категорию в меню или сайдбаре
2. Система отображает товары выбранной категории
3. Пользователь может выбрать подкатегорию
4. Система фильтрует товары

**Категории:**
- Электроника
- Одежда
- Бытовая техника
- Спорт и отдых
- Книги
- и т.д.

---

**Цель:** Найти конкретный товар по названию или описанию.

**Акторы:** Любой пользователь

## 3. Корзина покупок

### UC-3.1: Добавление товара в корзину

**Цель:** Добавить товар для последующей покупки.

**Акторы:** Любой пользователь (гость или авторизованный)

**Предусловия:**
- Товар доступен для покупки (inStock = true)

**Основной поток:**
1. Пользователь находится на странице товара или в каталоге
2. Пользователь выбирает количество (по умолчанию 1)
3. Пользователь нажимает "Добавить в корзину"
4. Система добавляет товар в корзину
5. Отображается уведомление "Товар добавлен"
6. Счётчик корзины обновляется

**Альтернативные потоки:**

**3.1a: Товар уже в корзине**
- Количество увеличивается на выбранное значение
- Или показывается предупреждение

**3.1b: Товар закончился**
- Кнопка "Добавить" неактивна
- Отображается "Нет в наличии"

**3.1c: Гость добавляет товар**
- Создаётся временная корзина в localStorage
- При авторизации корзина объединяется с серверной

---

### UC-3.2: Просмотр корзины

**Цель:** Посмотреть добавленные товары и сумму.

**Акторы:** Любой пользователь

**Основной поток:**
1. Пользователь нажимает иконку корзины в шапке
2. Открывается страница корзины
3. Отображается:
   - Список товаров с изображениями
   - Цена за единицу
   - Количество
   - Скидка (если есть)
   - Итоговая сумма за товар
   - Общая сумма корзины
   - Кнопка "Оформить заказ"

**Постусловия:**
- Пользователь видит содержимое корзины

---

### UC-3.3: Изменение количества товара

**Цель:** Увеличить или уменьшить количество товара.

**Акторы:** Любой пользователь с товарами в корзине

**Основной поток:**
1. Пользователь находится в корзине
2. Пользователь нажимает + или - рядом с количеством
3. Или вводит число вручную
4. Система обновляет количество
5. Пересчитывается сумма

**Альтернативные потоки:**

**3.3a: Количество = 0**
- Товар удаляется из корзины

**3.3b: Превышение остатка на складе (будущее)**
- Система показывает максимально доступное количество
- Устанавливает количество = остаток

---

### UC-3.4: Удаление товара из корзины

**Цель:** Убрать товар из корзины.

**Акторы:** Любой пользователь с товарами в корзине

**Основной поток:**
1. Пользователь нажимает иконку удаления или кнопку "Удалить"
2. Система запрашивает подтверждение (опционально)
3. Товар удаляется
4. Сумма пересчитывается
5. Отображается возможность "Отменить"

---

### UC-3.5: Очистка корзины

**Цель:** Удалить все товары из корзины.

**Акторы:** Любой пользователь

**Основной поток:**
1. Пользователь нажимает "Очистить корзину"
2. Система запрашивает подтверждение
3. Все товары удаляются
4. Корзина пуста

---

### UC-3.6: Применение промокода (будущее)

**Цель:** Получить скидку по промокоду.

**Акторы:** Любой пользователь

**Основной поток:**
1. Пользователь вводит промокод в специальное поле
2. Система проверяет промокод
3. Применяется скидка
4. Отображается размер скидки
5. Общая сумма пересчитывается

**Альтернативные потоки:**

**3.6a: Промокод недействителен**
- Система показывает ошибку
- Пользователь может ввести другой код

**3.6b: Промокод истёк**
- Система показывает "Срок действия промокода истёк"

## 4. Оформление и управление заказами

### UC-4.1: Оформление заказа

**Цель:** Создать заказ на товары из корзины.

**Акторы:** Авторизованный пользователь

**Предусловия:**
- Пользователь авторизован
- Корзина не пуста

**Основной поток:**
1. Пользователь нажимает "Оформить заказ" в корзине
2. Открывается страница оформления
3. Пользователь заполняет данные доставки:
   - Адрес доставки (обязательно)
   - Телефон (обязательно, можно использовать из профиля)
   - Email (обязательно, можно использовать из профиля)
4. Пользователь выбирает способ доставки:
   - Курьер (доставка на адрес)
   - Самовывоз (выбор пункта выдачи)
5. Пользователь выбирает способ оплаты:
   - Наличными при получении
   - Картой при получении
   - Онлайн оплата
6. Пользователь может добавить комментарий к заказу
7. Пользователь подтверждает заказ
8. Система создаёт заказ
9. Корзина очищается
10. Пользователь перенаправляется на страницу заказа

**Альтернативные потоки:**

**4.1a: Гость оформляет заказ**
- Пользователю предлагается войти или зарегистрироваться
- Или оформить заказ без регистрации (будущее)

**4.1b: Товар закончился**
- Система уведомляет о проблеме
- Предлагает удалить из заказа или ждать поступления

---

### UC-4.2: Просмотр истории заказов

**Цель:** Посмотреть все свои заказы.

**Акторы:** Авторизованный пользователь

**Предусловия:**
- Пользователь авторизован
- У пользователя есть хотя бы один заказ

**Основной поток:**
1. Пользователь переходит в "Мои заказы"
2. Отображается список заказов с:
   - Номером заказа
   - Датой создания
   - Статусом
   - Суммой
   - Количеством товаров
3. Заказы сортируются по дате (новые первыми)

---

### UC-4.3: Просмотр деталей заказа

**Цель:** Посмотреть подробную информацию о заказе.

**Акторы:** Авторизованный пользователь

**Основной поток:**
1. Пользователь нажимает на заказ в списке
2. Открывается страница заказа с:
   - Списком товаров
   - Ценами на момент заказа
   - Адресом доставки
   - Способом оплаты
   - Статусом заказа
   - Историей изменения статусов
   - Трек-номером (если есть)

---

### UC-4.4: Отмена заказа (будущее)

**Цель:** Отменить заказ до отправки.

**Акторы:** Авторизованный пользователь

**Предусловия:**
- Заказ в статусе "pending" или "processing"

**Основной поток:**
1. Пользователь открывает заказ
2. Пользователь нажимает "Отменить заказ"
3. Пользователь указывает причину отмены
4. Система отменяет заказ
5. Если оплата была онлайн - инициируется возврат

**Альтернативные потоки:**

**4.4a: Заказ уже отправлен**
- Кнопка отмены недоступна
- Предлагается вернуть товар после получения

---

### UC-4.5: Возврат товара (будущее)

**Цель:** Вернуть полученный товар.

**Акторы:** Авторизованный пользователь

**Предусловия:**
- Заказ в статусе "delivered"
- Не прошло 14 дней с получения

**Основной поток:**
1. Пользователь открывает заказ
2. Пользователь нажимает "Вернуть товар"
3. Пользователь выбирает товары для возврата
4. Пользователь указывает причину возврата
5. Пользователь выбирает способ возврата денег
6. Создаётся заявка на возврат
7. Возврат обрабатывается

### UC-4.6: Повтор заказа (будущее)

**Акторы:** Авторизованный пользователь

**Основной поток:**

**Акторы:** Авторизованный пользователь

## Приложение Б: Статусы заказа

```
pending ──────→ processing ──────→ shipped ──────→ delivered
    │               │                  │
    │               │                  │
    └───────────────┴──────────────────┴──→ cancelled
```

| Статус | Описание |
|--------|----------|
| pending | Заказ создан, ожидает обработки |
| processing | Заказ обрабатывается менеджером |
| shipped | Заказ отправлен клиенту |
| delivered | Заказ доставлен |
| cancelled | Заказ отменён |

## Приложение В: Статусы продукта

| Статус | Описание | Отображение |
|--------|----------|-------------|
